Visualizing and Demonstrating a Saddle Point

The problem of saddle points is a significant challenge in optimizing neural networks (and other complex functions). In high-dimensional spaces, saddle points are far more common than local minima.

A saddle point is a point where the gradient of the function is zero (or very close to zero), making it a critical point, but it is neither a local minimum nor a local maximum. If you move in one direction from a saddle point, the function value might increase, but if you move in another direction, it will decrease. This creates a "saddle" or "mountain pass" shape.

Why are they a problem for optimization?

  • Zero Gradient: Because the gradient is zero (or very small) at a saddle point, optimization algorithms like Gradient Descent (GD) can get "stuck" or significantly slow down. GD relies on the gradient to know which way to move downhill.
  • Slow Escape: Even if an optimizer eventually moves away, the region around a saddle point can be very flat, meaning the gradient remains small for a significant distance. This makes progress extremely slow.
  • High-Dimensional Space: In very high-dimensional loss landscapes (like those of deep neural networks), most critical points are saddle points, not local minima. This means optimizers frequently encounter them.

import numpy as np
import matplotlib.pyplot as plt
from mpl_toolkits.mplot3d import Axes3D

# --- 1. Define the Saddle Point Function and its Gradient ---
def saddle_function(x, y):
    """f(x, y) = x^2 - y^2"""
    return x**2 - y**2

def saddle_gradient(x, y):
    """Gradient of f(x, y) = (df/dx, df/dy) = (2x, -2y)"""
    df_dx = 2 * x
    df_dy = -2 * y
    return np.array([df_dx, df_dy])

# --- 2. Implement Gradient Descent ---
def gradient_descent(start_point, learning_rate, iterations, function_gradient):
    """
    Performs gradient descent on a 2D function.
    Returns a list of points (trajectory) and corresponding function values.
    """
    path = [np.array(start_point)]
    values = [saddle_function(start_point[0], start_point[1])]

    for i in range(iterations):
        current_x, current_y = path[-1]
        grad_x, grad_y = function_gradient(current_x, current_y)

        # Update rule: point = point - learning_rate * gradient
        new_x = current_x - learning_rate * grad_x
        new_y = current_y - learning_rate * grad_y
        
        new_point = np.array([new_x, new_y])
        path.append(new_point)
        values.append(saddle_function(new_x, new_y))

        # Stop if numerical instability (very large values) occurs
        if np.any(np.abs(new_point) > 1e10):
            print(f"Warning: Gradient exploded at iteration {i+1}. Stopping.")
            break

    return np.array(path), np.array(values)

# --- 4. Run Gradient Descent from Different Starting Points ---
learning_rate = 0.05
iterations = 200

# Scenario 1: Starting very close to the saddle point
start_point_1 = [0.01, 0.01] # Slightly off (0,0)
path1, values1 = gradient_descent(start_point_1, learning_rate, iterations, saddle_gradient)

# Scenario 2: Starting exactly on the saddle point (ideally no movement)
start_point_2 = [0.0, 0.0]
path2, values2 = gradient_descent(start_point_2, learning_rate, iterations, saddle_gradient)

# Scenario 3: Starting slightly off, but with one coordinate far from zero
start_point_3 = [0.01, 2.0] # Far in Y, close in X
path3, values3 = gradient_descent(start_point_3, learning_rate, iterations, saddle_gradient)

# Scenario 4: Starting slightly off, but with noise added at each step (simulating SGD or noise)
def gradient_descent_with_noise(start_point, learning_rate, iterations, function_gradient, noise_scale=0.01):
    path = [np.array(start_point)]
    values = [saddle_function(start_point[0], start_point[1])]

    for i in range(iterations):
        current_x, current_y = path[-1]
        grad_x, grad_y = function_gradient(current_x, current_y)

        # Add random noise to the gradient
        grad_x += np.random.normal(0, noise_scale)
        grad_y += np.random.normal(0, noise_scale)

        new_x = current_x - learning_rate * grad_x
        new_y = current_y - learning_rate * grad_y
        
        new_point = np.array([new_x, new_y])
        path.append(new_point)
        values.append(saddle_function(new_x, new_y))
        if np.any(np.abs(new_point) > 1e10):
            break
    return np.array(path), np.array(values)

start_point_4 = [0.01, 0.01] # Same as scenario 1
path4, values4 = gradient_descent_with_noise(start_point_4, learning_rate, iterations, saddle_gradient, noise_scale=0.05)

Explanation of the Output and Observations:

  1. 3D Surface Plot: This plot visually confirms the "saddle" shape of $f(x,y) = x^2 - y^2$. Notice how at $(0,0)$ (the red dot), the surface goes up along the X-axis and down along the Y-axis.

  2. Contour Plot with Gradient Descent Paths:

    • The contour lines show where the function value is constant. The origin $(0,0)$ is the saddle point.
    • Path 1 (Cyan): Starting slightly off $(0,0)$ (e.g., $(0.01, 0.01)$):
      • The gradient at $(0.01, 0.01)$ is $(0.02, -0.02)$, which is very small.
      • You'll see that GD makes very slow progress initially. It might drift slowly towards one of the "downhill" (Y-axis) directions but takes many iterations to get away from the flat region. Its function value also changes very slowly.
    • Path 2 (Pink): Starting exactly at $(0,0)$:
      • The gradient at $(0,0)$ is $(0,0)$.
      • The optimizer literally stays at $(0,0)$ for all iterations. Its function value remains $0$. This highlights the problem: if you land exactly on a saddle point with a perfectly deterministic optimizer, you're stuck.
    • Path 3 (Lime): Starting further away (e.g., $(0.01, 2.0)$):
      • The gradient at $(0.01, 2.0)$ is $(0.02, -4.0)$. The Y-component of the gradient is much larger.
      • You'll see GD quickly moves along the Y-axis (downhill) away from the saddle point. It only slows down as it approaches the flat region around $y=0$. This shows that if the optimizer starts with a significant "push" in a descending direction, it can escape quickly.
    • Path 4 (Orange, with Noise): Starting slightly off $(0,0)$ with added noise:
      • Even though it starts at the same initial point as Path 1, the added random noise to the gradient (mimicking Stochastic Gradient Descent's inherent randomness or a simple perturbation) helps the optimizer "jiggle" out of the flat saddle region.
      • It will make more substantial progress towards the "downhill" (Y-axis) direction much faster than Path 1, despite starting just as close to the saddle.

  3. Function Value Trajectories: This plot quantitatively shows the change in $f(x,y)$ over iterations for each path. You'll clearly see Path 2 remaining flat, Path 1 changing very slowly, and Paths 3 and 4 (especially Path 4 due to noise) showing a much more significant decrease in function value as they move away from the saddle point.

This example clearly illustrates why saddle points are problematic: they are flat regions where gradients are small, causing deterministic optimizers to slow down or get stuck. Stochastic methods (like SGD, which is what the noise simulates) can often navigate these regions more effectively due to their inherent randomness.
