The problem of saddle points is a significant challenge in optimizing the high-dimensional loss landscapes of deep neural networks.

While vanilla Gradient Descent (GD) can get "stuck" or slow down considerably around these flat regions, more advanced optimization techniques are designed to navigate them more effectively.

Let's illustrate how different optimizers handle a saddle point using the function $f(x, y) = x^2 - y^2$.

Optimizers to Tackle Saddle Points:

  1. Gradient Descent (GD) - Baseline:

    • Mechanism: Moves in the direction opposite to the gradient.
    • Saddle Point Behavior: Gets easily stuck or slows down significantly because the gradient is zero or very small at a saddle point.

  2. Gradient Descent with Momentum:

    • Mechanism: Accelerates GD in the relevant direction and dampens oscillations. It accumulates an exponentially decaying moving average of past gradients and adds it to the current gradient update.
      • $v\_t = \\beta v\_{t-1} + (1 - \\beta) \\nabla L(w\_t)$ (or just $v\_t = \\beta v\_{t-1} + \\nabla L(w\_t)$ depending on convention)
      • $w\_{t+1} = w\_t - \\alpha v\_t$
      • ($\\beta$ is the momentum coefficient, typically 0.9)
    • Saddle Point Behavior: The "inertia" provided by momentum can help push the optimizer past flat regions where the gradient is small, as it remembers past directions where there was a stronger gradient. It can effectively "roll" down the descending part of the saddle.

  3. Adam (Adaptive Moment Estimation):

    • Mechanism: Combines the benefits of Momentum and RMSprop (another adaptive learning rate optimizer). It calculates exponentially decaying averages of past gradients ($m\_t$, like momentum) and past squared gradients ($v\_t$, like RMSprop). It then uses these to adapt the learning rate for each parameter individually.
      • $m\_t = \\beta\_1 m\_{t-1} + (1 - \\beta\_1) \\nabla L(w\_t)$
      • $v\_t = \\beta\_2 v\_{t-1} + (1 - \\beta\_2) (\\nabla L(w\_t))^2$
      • $\\hat{m}\_t = m\_t / (1 - \\beta\_1^t)$ (bias correction for $m\_t$)
      • $\\hat{v}\_t = v\_t / (1 - \\beta\_2^t)$ (bias correction for $v\_t$)
      • $w\_{t+1} = w\_t - \\alpha \\frac{\\hat{m}\_t}{\\sqrt{\\hat{v}\_t} + \\epsilon}$
      • ($\\beta\_1$, $\\beta\_2$ are decay rates, $\\epsilon$ is a small number for numerical stability)
    • Saddle Point Behavior: Adam is very effective at navigating saddle points because:
      • The momentum component helps it push past flat regions.
      • The adaptive learning rate component, especially $\\sqrt{\\hat{v}\_t}$, scales the learning rate inversely proportional to the magnitude of past squared gradients. If gradients are consistently small in a certain direction (a flat region of a saddle), $\\hat{v}\_t$ will remain small, causing the effective learning rate in that direction to become larger, thus accelerating escape.

import numpy as np
import matplotlib.pyplot as plt
from mpl_toolkits.mplot3d import Axes3D

# --- 1. Define the Saddle Point Function and its Gradient ---
def saddle_function(x, y):
    """f(x, y) = x^2 - y^2"""
    return x**2 - y**2

def saddle_gradient(x, y):
    """Gradient of f(x, y) = (df/dx, df/dy) = (2x, -2y)"""
    df_dx = 2 * x
    df_dy = -2 * y
    return np.array([df_dx, df_dy])

# --- 2. Optimizer Implementations ---

class Optimizer:
    def __init__(self, learning_rate):
        self.learning_rate = learning_rate
        self.name = "Optimizer" # For plotting

    def update(self, current_point, gradient):
        raise NotImplementedError

    def get_path_and_values(self, start_point, iterations, func, grad_func):
        path = [np.array(start_point)]
        values = [func(start_point[0], start_point[1])]
        
        # Reset internal states for each run
        self._reset_state()

        for i in range(iterations):
            current_x, current_y = path[-1]
            grad = grad_func(current_x, current_y)
            
            # Update the point using the optimizer's specific logic
            new_point = self.update(path[-1], grad)
            
            # Simple check for numerical stability
            if np.any(np.abs(new_point) > 1e10) or np.any(np.isnan(new_point)):
                print(f"Warning: {self.name} exploded/NaN at iteration {i+1}. Stopping.")
                break

            path.append(new_point)
            values.append(func(new_point[0], new_point[1]))

        return np.array(path), np.array(values)

    def _reset_state(self):
        # Implement in subclasses if optimizer has internal state (e.g., momentum, Adam moments)
        pass

class GDOptimizer(Optimizer):
    def __init__(self, learning_rate):
        super().__init__(learning_rate)
        self.name = "Gradient Descent"

    def update(self, current_point, gradient):
        return current_point - self.learning_rate * gradient
    
    def _reset_state(self):
        pass # No state for vanilla GD

class MomentumOptimizer(Optimizer):
    def __init__(self, learning_rate, beta=0.9):
        super().__init__(learning_rate)
        self.beta = beta
        self.v = None # Velocity (momentum term)
        self.name = f"Momentum (={beta})"

    def _reset_state(self):
        self.v = None # Reset velocity for new runs

    def update(self, current_point, gradient):
        if self.v is None:
            self.v = np.zeros_like(gradient)
        
        # Velocity update: v = beta * v_prev + (1 - beta) * gradient (or just + gradient)
        # Using the simpler form for demonstration where beta is directly a decay factor
        self.v = self.beta * self.v + gradient 
        return current_point - self.learning_rate * self.v

class AdamOptimizer(Optimizer):
    def __init__(self, learning_rate, beta1=0.9, beta2=0.999, epsilon=1e-8):
        super().__init__(learning_rate)
        self.beta1 = beta1
        self.beta2 = beta2
        self.epsilon = epsilon
        self.m = None # First moment estimate (mean)
        self.v = None # Second moment estimate (variance)
        self.t = 0    # Timestep
        self.name = f"Adam (1={beta1}, 2={beta2})"

    def _reset_state(self):
        self.m = None
        self.v = None
        self.t = 0

    def update(self, current_point, gradient):
        self.t += 1
        if self.m is None:
            self.m = np.zeros_like(gradient)
            self.v = np.zeros_like(gradient)

        # Update biased first and second moment estimates
        self.m = self.beta1 * self.m + (1 - self.beta1) * gradient
        self.v = self.beta2 * self.v + (1 - self.beta2) * (gradient**2)

        # Bias-corrected first and second moment estimates
        m_hat = self.m / (1 - self.beta1**self.t)
        v_hat = self.v / (1 - self.beta2**self.t)

        # Update parameters
        return current_point - self.learning_rate * m_hat / (np.sqrt(v_hat) + self.epsilon)

--- Summary of Optimizer Behavior at Saddle Point ---
All optimizers started at (0.01, 0.01), very close to the saddle point (0,0).

1. Gradient Descent:
   - Initial gradient is very small (2*0.01, -2*0.01) = (0.02, -0.02).
   - You'll observe GD makes very little progress and largely stays stuck near the origin.
   - Its function value barely decreases from 0.
   - This illustrates why vanilla GD struggles with saddle points: flat gradients provide no strong direction.

2. Gradient Descent with Momentum:
   - Momentum accumulates past gradients.
   - Even though the initial gradient is small, momentum builds up, especially in the descending (Y) direction.
   - You'll see it eventually 'rolls' down the negative Y-axis of the saddle, making much better progress than vanilla GD.
   - Its function value decreases more significantly.

3. Adam Optimizer:
   - Adam combines momentum with adaptive learning rates.
   - The adaptive learning rate component increases the effective step size in directions where the gradient has been consistently small (like near a saddle point).
   - This allows Adam to escape the saddle point much more quickly and effectively than both vanilla GD and often Momentum.
   - It achieves the most significant decrease in function value, reaching a much lower (more negative) point on the saddle surface.
   - Note: Adam often requires a slightly higher overall learning rate compared to GD to show its full power.

Explanation of the Results:

  • Gradient Descent (GD) - (Blue Path):

    • As expected, the vanilla GD path stays very close to the origin. The initial gradients at (0.01, 0.01) are (0.02, -0.02), which are tiny. With a fixed learning rate, it makes minuscule steps and essentially gets "stuck" in the flat region of the saddle. Its function value hovers around zero.

  • Gradient Descent with Momentum - (Orange Path):

    • Momentum helps! Even though the initial gradient is small, the v (velocity) term starts accumulating the consistent (albeit small) push in the negative Y direction. This builds up over iterations, giving the optimizer "inertia" to overcome the flatness.
    • You'll see the orange path eventually picks up speed and moves significantly down the negative Y-axis, escaping the saddle point much more effectively than vanilla GD. Its function value drops notably.

  • Adam Optimizer - (Green Path):

    • Adam is generally the best performer here. It leverages both momentum and adaptive learning rates.
    • The m_hat term provides the momentum.
    • The v_hat term (based on squared gradients) keeps track of how "noisy" or "flat" the gradient has been for each dimension. Near a saddle point, the gradients are small, so v_hat will also be small. Dividing by sqrt(v_hat) effectively increases the learning rate in these flat dimensions, giving Adam a powerful kick to escape.
    • You'll observe Adam's path moves away from the saddle point the fastest, reaching a much more negative function value. This robust performance is why Adam (and its variants) are often default choices in deep learning.

This demonstration clearly shows how optimizers with mechanisms like momentum and adaptive learning rates are superior to vanilla gradient descent when navigating complex loss landscapes riddled with saddle points, a common scenario in deep learning.
