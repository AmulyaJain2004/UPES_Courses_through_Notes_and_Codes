Regularization methods are indispensable tools in a deep learning engineer's arsenal to combat overfitting.

As discussed, overfitting happens when a model learns the training data (including its noise) too well, leading to poor performance on unseen data. Regularization techniques achieve their goal by adding a penalty to the model's complexity or by making the learning process less reliant on specific neuron activations.

Here, we'll implement a simple feedforward neural network and demonstrate three common regularization techniques: L1, L2, and Dropout.

Scenario for Demonstration: Overfitting a Noisy Sine Wave

We'll create a synthetic dataset that clearly shows overfitting.

  • Data: A noisy sine wave.
  • Model: A relatively deep neural network (e.g., two hidden layers) with many neurons. This high capacity will make it prone to overfitting on limited data.
  • Demonstration:
    1. Train the model without regularization. Observe high training accuracy but poor generalization to unseen data.
    2. Train the model with L1 regularization.
    3. Train the model with L2 regularization.
    4. Train the model with Dropout.
    5. Compare their training and validation loss curves, as well as their predictive curves on the data.

Explanation of Regularization Techniques

  1. L1 Regularization (Lasso Regression):

    • Concept: Adds a penalty to the loss function equal to the absolute value of the weights multiplied by a regularization parameter ($\\lambda\_{L1}$).
      • $L\_{total} = L\_{MSE} + \\lambda\_{L1} \\sum |w|$
    • Effect: Encourages sparsity in the weights, meaning some weights will be driven to exactly zero. This effectively performs feature selection, making the model simpler by effectively removing certain connections.
    • In Code:
      • l1_lambda parameter in __init__.
      • In _calculate_regularization_loss(), self.l1_lambda * np.sum(np.abs(W)) is added to the loss.
      • In backward(), self.l1_lambda * np.sign(W) is added to the weight gradient (dW). The derivative of $|w|$ is $\\text{sign}(w)$.
    • Observation (Plots): You'll see the validation loss stabilize earlier or be lower than the "No Reg" case, and the prediction curve will be smoother, less wavy, and closer to the true sine wave, especially if the l1_lambda is well-tuned.

  2. L2 Regularization (Ridge Regression / Weight Decay):

    • Concept: Adds a penalty to the loss function equal to the square of the weights multiplied by a regularization parameter ($\\lambda\_{L2}$).
      • $L\_{total} = L\_{MSE} + \\lambda\_{L2} \\sum w^2$
    • Effect: Encourages smaller weights, distributing weight more evenly across features. It prevents any single weight from becoming too large and dominating the learning, thus leading to a smoother decision boundary and better generalization. It does not force weights to be exactly zero.
    • In Code:
      • l2_lambda parameter in __init__.
      • In _calculate_regularization_loss(), self.l2_lambda * np.sum(W**2) is added to the loss.
      • In backward(), self.l2_lambda * 2 * W is added to the weight gradient (dW). The derivative of $w^2$ is $2w$.
    • Observation (Plots): Similar to L1, L2 regularization will show better validation loss and a smoother prediction curve compared to the overfit model. It's generally preferred over L1 for non-sparse solutions.

  3. Dropout:

    • Concept: During each training iteration, a random subset of neurons (and their connections) in a layer are temporarily "dropped out" (set to zero). This means those neurons do not contribute to the forward pass or backpropagation in that specific iteration.
    • Effect:
      • Reduces Co-adaptation: Prevents neurons from relying too much on specific other neurons. Each neuron learns to be more robust and perform well independently.
      • Ensemble Effect: It can be thought of as training a slightly different "thinned" neural network in each iteration, effectively creating an ensemble of many different networks. At test time, all neurons are used, but their outputs are scaled down by the dropout rate (or up during training, as implemented here) to approximate the average output of these "ensemble" networks.
    • In Code:
      • dropout_rate parameter in __init__.
      • In forward(), when is_training=True:
        • mask = (np.random.rand(*A.shape) > self.dropout_rate): Creates a boolean mask to drop neurons.
        • A = A * mask / (1 - self.dropout_rate): Applies the mask. The division by (1 - self.dropout_rate) is crucial for scaling. It ensures that the expected sum of outputs from the layer remains the same during training as it would be at test time (when no dropout is applied).
      • In backward(), the same mask is applied to the gradients (d_A_curr) to ensure that dropped neurons do not receive updates.
      • During predict() (or is_training=False), no dropout is applied.
    • Observation (Plots): Dropout can be very effective in reducing overfitting. The training loss might be slightly higher than for an overfit model (because the model is intentionally being "hindered" during training), but the validation loss will be significantly better, and the prediction curve will be much smoother and generalize well.

# --- 2. Activation Functions and Derivatives ---
def relu(x):
    return np.maximum(0, x)

def relu_derivative(x):
    # This correctly returns 1 for x > 0 and 0 for x <= 0.
    # The warning comes if 'x' itself already contains NaN.
    return np.where(x > 0, 1, 0)

# --- 3. Loss Function (Mean Squared Error) ---
def mse_loss(y_pred, y_true):
    return np.mean((y_pred - y_true)**2)

# --- 4. Neural Network Class with Regularization Options ---
class NeuralNetworkRegularized:
    def __init__(self, input_size, hidden_layer_sizes, output_size, learning_rate=0.01,
                 l1_lambda=0.0, l2_lambda=0.0, dropout_rate=0.0):
        
        self.learning_rate = learning_rate
        self.l1_lambda = l1_lambda # L1 regularization strength
        self.l2_lambda = l2_lambda # L2 regularization strength
        self.dropout_rate = dropout_rate # Dropout probability (0 = no dropout)

        self.weights = []
        self.biases = []
        self.Zs = [] # Store pre-activation values
        self.As = [] # Store post-activation values (except final output)
        self.dropout_masks = [] # Store masks for dropout

        # Initialize weights and biases for each layer
        prev_size = input_size
        for i, h_size in enumerate(hidden_layer_sizes):
            # He initialization for ReLU
            self.weights.append(np.random.randn(prev_size, h_size) * np.sqrt(2.0 / prev_size))
            self.biases.append(np.zeros((1, h_size)))
            prev_size = h_size
            
        # Output layer weights and biases
        self.weights.append(np.random.randn(prev_size, output_size) * np.sqrt(2.0 / prev_size))
        self.biases.append(np.zeros((1, output_size)))

    def forward(self, X, is_training=True):
        self.Zs = []
        self.As = [X] # A0 is the input X
        self.dropout_masks = [] # Reset masks for each forward pass

        # Hidden Layers
        for i in range(len(self.weights) - 1): # Loop through hidden layers
            Z = np.dot(self.As[-1], self.weights[i]) + self.biases[i]
            A = relu(Z)
            
            # --- Dropout ---
            if is_training and self.dropout_rate > 0:
                mask = (np.random.rand(*A.shape) > self.dropout_rate) / (1 - self.dropout_rate)
                A = A * mask
                self.dropout_masks.append(mask)
            else:
                # If not training (e.g., validation/prediction), no dropout.
                # Dropout already scales outputs during training by (1-dropout_rate),
                # so no scaling needed here.
                # We still need a placeholder mask if we iterate through dropout_masks in backward
                self.dropout_masks.append(np.ones(A.shape)) # Ensure mask is appended even when not training

            self.Zs.append(Z)
            self.As.append(A)
            
        # Output Layer (linear activation for regression)
        Z_out = np.dot(self.As[-1], self.weights[-1]) + self.biases[-1]
        A_out = Z_out # Linear activation
        
        self.Zs.append(Z_out) # Z for output layer is stored
        self.As.append(A_out) # A for output layer is stored

        return A_out

    def backward(self, X, y_true, y_pred):
        # Initial error at output layer
        d_A_out = y_pred - y_true # dLoss/d(y_pred) for MSE

        # Gradients for output layer
        d_Z_out = d_A_out # Linear activation derivative is 1

        dW_out = np.dot(self.As[-2].T, d_Z_out) # (A_L-1).T @ dZ_L
        db_out = np.sum(d_Z_out, axis=0, keepdims=True)

        # --- Add Regularization Gradients to Output Layer Weights ---
        if self.l1_lambda > 0:
            dW_out += self.l1_lambda * np.sign(self.weights[-1])
        if self.l2_lambda > 0:
            dW_out += self.l2_lambda * 2 * self.weights[-1] # Derivative of W^2 is 2W

        # Update output layer parameters
        self.weights[-1] -= self.learning_rate * dW_out
        self.biases[-1] -= self.learning_rate * db_out

        # Backpropagate through hidden layers
        d_Z_curr = d_Z_out
        for i in reversed(range(len(self.weights) - 1)):
            # Error propagated back from next layer to current layer's activation
            d_A_curr = np.dot(d_Z_curr, self.weights[i+1].T)

            # --- Dropout Gradient ---
            # Apply mask to propagate gradient only through active neurons
            # The mask was already scaled by (1 - dropout_rate) in forward
            # if self.dropout_rate > 0: # This check is already implicitly handled by how masks are stored
            d_A_curr = d_A_curr * self.dropout_masks[i]
            
            # Error propagated through current layer's activation derivative
            # This is the line causing the warning if d_A_curr or self.Zs[i] contains NaNs
            d_Z_curr = d_A_curr * relu_derivative(self.Zs[i])

            dW_curr = np.dot(self.As[i].T, d_Z_curr)
            db_curr = np.sum(d_Z_curr, axis=0, keepdims=True)

            # --- Add Regularization Gradients to Current Layer Weights ---
            if self.l1_lambda > 0:
                dW_curr += self.l1_lambda * np.sign(self.weights[i])
            if self.l2_lambda > 0:
                dW_curr += self.l2_lambda * 2 * self.weights[i]

            # Update current layer parameters
            self.weights[i] -= self.learning_rate * dW_curr
            self.biases[i] -= self.learning_rate * db_curr

    def _calculate_regularization_loss(self):
        reg_loss = 0.0
        if self.l1_lambda > 0:
            for W in self.weights:
                reg_loss += self.l1_lambda * np.sum(np.abs(W))
        if self.l2_lambda > 0:
            for W in self.weights:
                reg_loss += self.l2_lambda * np.sum(W**2)
        return reg_loss

    def train(self, X_train, y_train, X_val, y_val, epochs):
        train_loss_history = []
        val_loss_history = []

        for epoch in range(epochs):
            # Training Pass
            y_train_pred = self.forward(X_train, is_training=True)
            mse_train_loss = mse_loss(y_train_pred, y_train)
            
            # Total loss includes regularization for training
            total_train_loss = mse_train_loss + self._calculate_regularization_loss()
            train_loss_history.append(total_train_loss)
            
            self.backward(X_train, y_train, y_train_pred)

            # Validation Pass (no dropout or regularization in loss calculation)
            y_val_pred = self.forward(X_val, is_training=False)
            val_loss = mse_loss(y_val_pred, y_val)
            val_loss_history.append(val_loss)

            if epoch % (epochs // 10) == 0:
                print(f"Epoch {epoch:4d} | Train Loss: {total_train_loss:.6f} | Val Loss: {val_loss:.6f}")
        
        print(f"Epoch {epochs-1:4d} | Train Loss: {total_train_loss:.6f} | Val Loss: {val_loss_history[-1]:.6f}")
        return train_loss_history, val_loss_history

    def predict(self, X):
        # For prediction, dropout is not applied (is_training=False)
        return self.forward(X, is_training=False)

Explanation of Regularization Techniques

  1. L1 Regularization (Lasso Regression):

    • Observation (Plots): You'll see the validation loss stabilize earlier or be lower than the "No Reg" case, and the prediction curve will be smoother, less wavy, and closer to the true sine wave, especially if the l1_lambda is well-tuned.

  2. L2 Regularization (Ridge Regression / Weight Decay):

    • Observation (Plots): Similar to L1, L2 regularization will show better validation loss and a smoother prediction curve compared to the overfit model. It's generally preferred over L1 for non-sparse solutions.

  3. Dropout:

    • Observation (Plots): Dropout can be very effective in reducing overfitting. The training loss might be slightly higher than for an overfit model (because the model is intentionally being "hindered" during training), but the validation loss will be significantly better, and the prediction curve will be much smoother and generalize well.

By running this code, you'll clearly see how the "No Regularization" model fits the noise in the training data (wobbly prediction line, large gap between train and validation loss), while the regularization techniques force the model to learn a smoother, more generalized function that fits both training and validation data better.
